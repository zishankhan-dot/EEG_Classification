Downloading Data:
created a data downloading script ./scripts/download_data.py which downloads the data from the provided link and saves it in the ../data/raw/ directory. for the purpose of this project i have targeted first 20 subjects and 3 recordings for each subject(04,08,12).

Data Exploration And Analysis:
Then, I created a data loading script ./src/data.py, which loads the data using mne library and performs some basic preprocessing steps such as filtering the data between 0.5 and 40 Hz. and comparing the raw and filtered data by plotting three graphs one for raw data (time domain) and another for power spectral density (frequency domain) and the last one with color map(c3,c4,cz) to understand how c3,c4 and cz are behaving opposite to each other.

Channel Selection:
the dataset has 64 channels across the scalp. after plotting all channels on the scalp using the mne standard_1005 montage i identified that motor imagery is processed by the motor cortex which sits along the central strip of the brain. C3 is placed over the left motor cortex and activates for right hand imagery, C4 is over the right motor cortex and activates for left hand imagery, Cz sits at the center midline. these three were selected because they are directly over the region that should show the strongest signal for left vs right hand motor imagery. the edf files store channel names with dots like C3. so those were stripped on load to match the montage naming convention.

Band Selection:
two frequency bands were investigated, mu (8-13 Hz) and beta (13-30 Hz). both are generated by the motor cortex at rest and both suppress when someone imagines a movement. this suppression is called ERD (event related desynchronization). when the motor cortex activates it goes quiet in these bands rather than louder, which is counterintuitive but is the actual detectable signature used for motor imagery classification. i verified this by computing PSD per condition (T0=rest, T1=left, T2=right) across C3, C4 and Cz separately. the plots showed separation between rest and imagery in the mu range around 8-13 Hz. i also ran a morlet wavelet analysis with a -2s to +5s window around each event and a pre-stimulus baseline from -2s to -0.5s to visualize ERD/ERS patterns over time and frequency. the ERD (blue) was visible during the imagery window and the ERS rebound (red) appeared after the event on some subjects. i also plotted C3 minus C4 power difference to check lateralization - if left hand imagery produces more ERD on C4 and right hand produces more on C3, the difference map should flip between T1 and T2. it did on high-accuracy subjects but was weak or absent on noisy ones.

Feature Extraction:
for each 4-second epoch i computed Welch PSD and extracted log band power in the mu band and beta band for each of the 3 channels (C3, C4, Cz). this gives 6 features per epoch (3 channels x 2 bands). i also tried relative band power where each band power is divided by total power across the full frequency range to normalize it. absolute features consistently outperformed relative features. i also tried combining both into 12 features per epoch but this did not improve results over using absolute features alone. the feature distributions for T1 and T2 overlap significantly which explains why classification is difficult. there is no clean separation in any single feature.

Model Training:
trained four classifiers - LDA, SVM, Random Forest and MLP - using scikit-learn. the training pipeline loads the 25 selected subjects, concatenates all their runs, filters (8-30 Hz), creates epochs (tmin=0, tmax=4), extracts the 6 band power features and trains each model with StratifiedKFold cross-validation. each trained model is saved as a joblib bundle in ./models/ containing the model object, cross-val accuracy and cross-val F1 score. LDA performed best overall in the cross-subject setup.

Evaluation - Cross-Subject:
the cross-subject model was trained on all 25 selected subjects pooled together. evaluation with StratifiedKFold gave average accuracy of 0.663 for LDA, 0.633 for MLP, 0.614 for SVM and 0.589 for RF. i also tried GroupKFold where each fold held out entire subjects - this gave lower scores since the model was tested on subjects it had never seen. both approaches confirmed that cross-subject generalization is the core difficulty.

Evaluation - Within-Subject:
within-subject evaluation trains and tests on the same subject using stratified splits. this gave better numbers - LDA averaged 0.704, MLP 0.687, RF 0.670, SVM 0.667 across the 25 subjects. only 2 subjects crossed 89% accuracy (subject 29 and subject 72). most subjects fell between 0.65 and 0.80. these numbers are optimistic because the model is evaluated on the same individual it was trained on, but they show that per-person calibration would significantly improve results.

Accuracy Issues and What Was Tried:
the cross-subject model stayed around 58-66% despite several attempts to improve it.

first issue was the epoch window. initially used tmin=2, tmax=6 to skip the onset transient and focus on the steady-state imagery period. this caused all epochs to be dropped because the PhysioNet trial is only 4 seconds long and tmax=6 extends past the end of the trial. changed to tmin=0, tmax=4.

second attempt was processing each run (R04, R08, R12) separately instead of concatenating all runs per subject before epoching. concatenation creates artificial boundaries between runs which can affect filtering and epoching. created a separate data2.py file for this. results were similar either way.

third attempt was subject filtering. started with first 20 subjects and got poor results. ran per-subject cross-validation on all 25 subjects that were later identified as good. training on subjects with per-subject accuracy below 65% introduced noise that hurt the cross-subject model. narrowing to 25 subjects above 65% improved results slightly.

fourth attempt was trying relative vs absolute features and combining both. absolute features were best across all models. the difference was most visible for LDA (0.663 absolute vs 0.554 relative).

fifth attempt was CSP (common spatial patterns) which is the standard spatial filter used in motor imagery BCI research. ran it within-subject but it did not produce meaningful improvement given the inter-subject variance in this dataset.

sixth attempt was trying different cross-validation strategies including GroupKFold, StratifiedKFold and within-subject stratified split. results varied by method but none pushed the cross-subject number above 66%.

none of these resolved the core issue. the cross-subject model is fundamentally limited by the variance between individuals.

Why The Dataset Is Difficult:
skull thickness varies between people. a thicker skull dampens and spatially spreads the electrical signal before it reaches the electrode surface. two people performing the same mental task can produce very different amplitude and spatial patterns just because of skull geometry, not because their brain activity is different.

electrode impedance and contact quality also varies between subjects and even between sessions for the same subject. small changes in how the cap is placed shift which neurons are closest to each electrode.

the mental strategy each person uses for motor imagery is uncontrolled. some people visualize the movement visually, others feel the kinesthetic sensation of movement. these activate different neural circuits and produce different frequency patterns. there is no way to detect or control for this from the signal alone.

high inter-subject variance means a feature that cleanly separates T1 and T2 for one person may be completely uninformative for another. the 6 band power features capture average frequency content but miss fine spatial and temporal structure that is highly individual.

using only 3 channels from 64 also limits spatial resolution. a full spatial filter over all 64 channels with subject-specific calibration could capture more discriminative patterns but cannot generalize across subjects without individual recording sessions for fine-tuning.

Streamlit App:
built a full interactive web app in app.py using streamlit. the sidebar has a data source section where the user can either upload a PhysioNet EDF file (runs 4, 8 or 12 only) or click a button to load a demo file which downloads subject 29 run 4 automatically using mne.datasets.eegbci without prompting the user. once a file is loaded the app shows four metric cards (sampling frequency, total channels, recording duration, annotations), three side-by-side plots (raw signal with event markers, raw vs filtered comparison, motor cortex sensor map with C3/C4/Cz highlighted), and a graphviz pipeline diagram showing the full processing flow. the second half of the app is the model inference section where the user selects a model (LDA, SVM, RF, MLP) and clicks run prediction. this runs the full pipeline on the uploaded file and returns accuracy, F1 score, a per-epoch prediction table with color coded correct/incorrect column, a confusion matrix and a prediction timeline showing ground truth vs predicted label for every epoch as a horizontal bar chart.

Deployment:
deployed the app on Streamlit Cloud connected to the GitHub repository. the trained model files in ./models/ were committed to the repo so Streamlit Cloud can load them at runtime. the mne.datasets.eegbci download for the demo subject uses a temp directory path with update_path=False to avoid any interactive prompts that would hang on a cloud server. requirements are listed in requirements.txt.
